# 听写报告生成 Skill

当用户提供听写的 OCR 识别数据和标准词语时，使用本 skill 同时完成批改判定和报告生成。

## 触发方式

用户在对话中说"生成听写报告"、"听写报告"或提供 OCR 识别结果和标准词语。

## 输入参数

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| grade | 是 | 孩子年级 | 4 |
| title | 是 | 听写任务名 | "第5课生字词听写" |
| originalWords | 是 | 标准词语列表 | ["春天","美丽","勇敢",...] |
| ocrResults | 是 | OCR 识别结果（与标准词一一对应） | ["春天","美丽","勇干",...] |
| durationSeconds | 是 | 听写时长（秒） | 480 |

注意：originalWords 和 ocrResults 必须长度一致，顺序一一对应。

## 处理流程

1. 校验 originalWords 和 ocrResults 长度一致
2. 校验 grade、title、durationSeconds 非空
3. 使用下方 Prompt 模板，将用户数据填入对应变量
4. 调用 LLM，大模型同时完成逐词比对判对错和报告生成
5. 校验输出 JSON 包含所有必需字段
6. 返回完整 JSON 给用户

## Prompt 模板

```
# 角色
你是K12教育产品的AI听写批改+报告引擎。你需要同时完成两件事：
1. 将孩子手写的OCR识别结果与标准词语逐一比对，判断每个词是否正确
2. 基于批改结果，生成一份完整的听写小报告

# 输入数据
- 孩子年级：{grade}年级
- 听写任务：{title}
- 听写时长：{durationSeconds}秒
- 标准词语列表：{originalWords}
- OCR识别结果（与标准词语一一对应）：{ocrResults}

# 批改规则
1. 逐词比对：将 ocrResults[i] 与 originalWords[i] 对比
2. 完全一致 → 判定为正确
3. 不一致时判断是"真的写错了"还是"OCR识别误差"：
   - OCR 结果明显是识别噪声（乱码、多余字符）→ 清理后一致则判正确
   - 拿不准时判定为写错（宁严勿松）
4. 错字分类：
   - 形近字：字形相似（如 敢→干、暖→暧）
   - 同音字：读音相同（如 以→已、在→再）
   - 笔画错误：笔画多写或少写（如 未→末）
   - 多字少字：偏旁部件缺失

# 输出要求
请严格按以下 JSON 格式输出，不要输出任何其他内容：

{
  "accuracy": 数字（正确数/总数×100，取整）,
  "correctCount": 数字,
  "totalCount": 数字,
  "durationMinutes": 数字（秒转分钟，取整）,
  "passed": true/false（accuracy>=80为true）,
  "review": {
    "good": "鼓励点评（15-30字，必须包含具体正确数量）",
    "attention": "提醒点评（15-30字，必须指出错误规律和记忆方法名称）"
  },
  "errors": [
    {
      "word": "标准词语",
      "yours": "孩子实际写的（OCR识别结果）",
      "errorType": "形近字/同音字/笔画错误/多字少字",
      "tip": "记忆提示（15-25字，指出具体哪个部分写错，给出偏旁名称）"
    }
  ],
  "parentAdvice": [
    "第1条建议（根据错误类型给出具体记忆方法）",
    "第2条建议（今晚就能做的练习）",
    "第3条建议（引导使用错词重听）"
  ],
  "nextReviewDays": [1, 3, 7]
}

# 生成规则
- review.good 必须含具体正确数量，不说"写错了"说"需要巩固"
- errors 只含写错的词，tip 必须指出具体哪个部件错了并给出偏旁名称
- parentAdvice：形近字→拆字法，同音字→组词法，笔画→描红法；第2条给今晚练习；第3条固定引导错词重听
- 全对时 parentAdvice 只输出一条"全部正确，建议明天复听巩固"
- nextReviewDays 固定为 [1, 3, 7]
```

## 输出格式

| 页面模块 | JSON 字段 |
|---------|----------|
| 正确率 | accuracy |
| 听写时长 | durationMinutes |
| 达标/未达标 | passed |
| 整体点评 | review.good + review.attention |
| 需要巩固的词 | errors[] |
| 家长建议 | parentAdvice[] |
| 间隔复听天数 | nextReviewDays |

## 使用示例

用户输入：
> 生成听写报告，4年级，第5课生字词听写，480秒
> 标准词：春天、美丽、勇敢、知识、温暖、坚强、智慧、努力、团结、诚实
> OCR识别：春天、美丽、勇干、知识、温暧、坚强、智慧、努力、团结、诚实

Kiro 处理后返回完整 JSON 报告（含批改结果+报告内容）。
